When it was time to decide which major to choose in college, it was a no-brainer, Computer Science. During my second semester of college, my friends were talking about going to a hackathon, HackBU, where we would spend 24 hours coding. During the event I attended various programming workshops and Tech Talks, which was a great way for me to develop additional skills in my field and be exposed to this kind of programming environment.
One of my favorite projects was done at Cooper Union during a 24-hour hackathon. Our project was to extract tweets from the Twitter API using Java and exporting those tweets and other data to a JSON file. Using JavaScript, HTML, CSS, we created a tribunal system that acts as a community policing program. My job in this team was to use Java and the Twitter API to query their database for the most recent tweet containing the string we were searching for. Additionally, I worked on the front-end of our program, using Javascript, it received data from our JSON file and displayed that information on our page along with its interface. During Hack Cooper, we were presented with a task to do something related to Hack Harassment. My team and I realized that we could improve the social networking experience by giving incentives to report offensive users. In order to increase reports on disrespectful users, we decided to game-ify our project by giving participants badges to add to their profile. Due to the rise of cyberbullying on social networks, finding a way to help the community "remove" offensive posts was a great way to contribute to the Hack Harassment movement.

Vision 2.0 KEMP Bot

Worked under Jon Malek, Systems Architect, along with a team of senior software developers, system integrators, and quality assurance engineers on the development of the KEMP 360 Vision 2.0 Slack bot.

Moved hard-coded communication platform to an advanced NLP platform by integrating NLP to KEMP Bot using Microsoft Luis which calculates the score for each possible user intent and outputs the top scoring intent. The intents are trained using the LUIS API in order to achieve an accurate result. The top scoring intent is called and the bot performs the intended action.

Used Chart.js to graph user database queries within a specfic time range. For this project, we used InfluxDB for storing customer data. A simple database call outputs the requested JSON data which is parsed and sent to Chart.js to render and save a temporary image of the graph that is sent to the Slack API as an attachment for the user.

PCFG Parser

Part 1:

PCFG Induction

Run: python3 pcfg_induction.py

Reads from the training data file, 'train.trees'. Writes to several files:

unary_prob.txt Unary rules involving lexical items. For example, they are in the format, VB List 0.01256281407035176. Total unary rules: 397 for train.trees

binary_prob.txt Binary rules involving POS or syntactic categories. For example, they are in the format, TOP S_VP PUNC 0.2049335863377609. Total binary rules: 393 for train.trees

top_unary.txt Lists the top 10 most frequent unary rules. Output for train.trees: ('PUNC', '.') 383 ('TO', 'to') 268 ('IN', 'from') 243 ('NNS', 'flights') 169 ('PUNC', '?') 144 ('DT', 'the') 137 ('VB', 'Show') 94 ('NP_PRP', 'me') 90 ('WDT', 'What') 85 ('NN', 'flight') 81

top_binary.txt Lists the top 10 most frequent binary rules. Output for train.trees: ('PP', ('IN', 'NP_NNP')) 263 ('PP', ('IN', 'NP')) 222 ('PP', ('TO', 'NP_NNP')) 175 ('NP', ('NNP', 'NNP')) 151 ('TOP', ('SBARQ', 'PUNC')) 119 ('TOP', ('FRAG_NP', 'PUNC')) 116 ('TOP', ('S_VP', 'PUNC')) 108 ("S_VP'", ('VB', 'NP_PRP')) 84 ('S_VP', ("S_VP'", 'NP')) 81 ('NP', ('DT', 'NN')) 79

I used Laplace smoothing for unary rules and MLE probabilities for binary rules.

Part 2:

Run: python3 cky_parser.py test.txt

This program will read from 'unary_prob.txt' and 'binary_prob.txt' and build the data structures needed for cky parse. After reading from file, The program will read each sentence from 'test.txt' and cky parse each sentence, building the highest probability tree used by evalb.py.

For part this part of the assignment, I was unable to construct a proper tree with my backpointers. The output of the cky parse was a dictionary of tuples that contained triples as a value. For example, backpointers looked like {(0, 2, 'NP'): (1, 'DT', 'NN'), (0, 2, "NP'"): (1, 'DT', 'NN'), (5, 7, 'NP'): (6, 'CD', 'RB'), (5, 7, "NP'"): (6, 'CD', 'RB'),...} The way this was formated: back(begin, end, A) = (split, B, C) according to the algorithm on the pdf. The root node of the tree was back[(0, len(words), 'TOP')]. The way I am supposed to construct the tree is to start at the root node and create two subtrees, left and right. The left subtree is back[(0, len(words), 'S')] and the right subtree is back[(8, len(words), 'PUNC')]. However, I was could not get the entire tree built, so I printed out the backpoints for each sentence as well as the root node and its value.

Part 3:

A tree was needed to run evalb.py so I could compare my output.trees with test.trees. This part was not possible without a full tree.
